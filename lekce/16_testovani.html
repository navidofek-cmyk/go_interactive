Lekce 16: 🧪 Testování — go test

⭐⭐⭐⭐ Těžší ⏱ ~35 minut Lekce 16 / 16

Co je testování?

Testování je jako kontrolní otázky pro tvůj program. Napíšeš test: "Když zavolám funkci Secti(3, 5), musím dostat 8." Pak spustíš go test a Go ti řekne, jestli vše funguje správně.

Proč testovat? Představ si, že děláš velký LEGO hrad. Po každé změně zkontroluj, jestli se nezhroutil. Testy dělají totéž pro kód — hlídají, že nic nerozbíjíš.

✈️ Testy = kontrolní seznam pilota před letem. Pilot zkontroluje každou věc: "Motor OK? ✅ Palivo OK? ✅ Dveře OK? ✅". Teprve pak vzlétne. Programátor zkontroluje: "Secti(3,5)=8? ✅ Vydel(10,2)=5? ✅"

Pravidla testů v Go:

  • Soubor musí mít příponu _test.go
  • Funkce musí začínat Test
  • Parametr je vždy t *testing.T
  • Spustíš příkazem go test ./...
Struktura projektu a výstup go test:
Soubory v projektu: matematika.go ← funkce které píšeš matematika_test.go ← testy pro ty funkce Spuštění: $ go test ./... --- PASS: TestSecti (0.00s) ✅ test prošel --- FAIL: TestVydel (0.00s) ❌ test selhal Got 3, want 5
1. Soubor s funkcemi
matematika.go
// Soubor: matematika.go
// Toto jsou funkce, které budeme testovat

package matematika

import "fmt"

// Secti vrátí součet dvou čísel
func Secti(a, b int) int {
    return a + b
}

// Vydel vrátí podíl — nebo chybu pokud dělíme nulou
func Vydel(a, b int) (int, error) {
    if b == 0 {
        return 0, fmt.Errorf("nelze dělit nulou")
    }
    return a / b, nil
}

// JeSude vrátí true pokud je číslo sudé
func JeSude(n int) bool {
    return n%2 == 0
}

// Maximum vrátí větší ze dvou čísel
func Maximum(a, b int) int {
    if a > b {
        return a
    }
    return b
}
2. Testovací soubor
matematika_test.go
// Soubor: matematika_test.go
// Spusť: go test ./...     nebo: go test -v ./...

package matematika

import "testing"

// Základní test
func TestSecti(t *testing.T) {
    vysledek := Secti(3, 5)
    if vysledek != 8 {
        t.Errorf("Secti(3, 5) = %d, chci 8", vysledek)
    }
}

// Table-driven test — testujeme víc příkladů najednou!
func TestSectiVice(t *testing.T) {
    priklady := []struct {
        a, b     int
        ocekavam int
    }{
        {1, 2, 3},
        {0, 0, 0},
        {-5, 5, 0},
        {100, 200, 300},
        {-3, -7, -10},
    }

    for _, p := range priklady {
        vysledek := Secti(p.a, p.b)
        if vysledek != p.ocekavam {
            t.Errorf("Secti(%d, %d) = %d, chci %d",
                p.a, p.b, vysledek, p.ocekavam)
        }
    }
}

func TestVydel(t *testing.T) {
    // Správné dělení
    v, err := Vydel(10, 2)
    if err != nil {
        t.Fatalf("Vydel(10,2) neočekávaná chyba: %v", err)
    }
    if v != 5 {
        t.Errorf("Vydel(10,2) = %d, chci 5", v)
    }

    // Dělení nulou musí vrátit chybu
    _, err = Vydel(5, 0)
    if err == nil {
        t.Error("Vydel(5,0) mělo vrátit chybu, ale vrátilo nil")
    }
}

func TestJeSude(t *testing.T) {
    // Subtesty — pojmenované skupiny testů
    t.Run("sudá čísla", func(t *testing.T) {
        for _, n := range []int{0, 2, 4, 100, -6} {
            if !JeSude(n) {
                t.Errorf("%d mělo být sudé", n)
            }
        }
    })

    t.Run("lichá čísla", func(t *testing.T) {
        for _, n := range []int{1, 3, 7, 99, -5} {
            if JeSude(n) {
                t.Errorf("%d nemá být sudé", n)
            }
        }
    })
}

func TestMaximum(t *testing.T) {
    if Maximum(3, 7) != 7 {
        t.Error("Maximum(3,7) mělo vrátit 7")
    }
    if Maximum(10, 2) != 10 {
        t.Error("Maximum(10,2) mělo vrátit 10")
    }
    if Maximum(5, 5) != 5 {
        t.Error("Maximum(5,5) mělo vrátit 5")
    }
}
3. Jak spustit testy
Příkazy v terminálu:
$ go test ./... — spustí všechny testy $ go test -v ./... — podrobný výpis (verbose) $ go test -run TestSecti ./... — jen konkrétní test $ go test -cover ./... — kolik kódu je otestováno
Výsledky testů
📺 Příklad úspěšných testů (go test -v ./...):
$ go test -v ./... === RUN TestSecti --- PASS: TestSecti (0.00s) === RUN TestSectiVice --- PASS: TestSectiVice (0.00s) === RUN TestVydel --- PASS: TestVydel (0.00s) === RUN TestJeSude === RUN TestJeSude/sudá_čísla === RUN TestJeSude/lichá_čísla --- PASS: TestJeSude (0.00s) === RUN TestMaximum --- PASS: TestMaximum (0.00s) PASS ok matematika 0.003s
📺 Příklad selhání testu:
$ go test -v ./... === RUN TestSecti matematika_test.go:9: Secti(3, 5) = 9, chci 8 --- FAIL: TestSecti (0.00s) FAIL FAIL matematika 0.002s

Úkoly na procvičení

  1. Přidej funkci Nasob(a, b int) int do matematika.go a napiš test TestNasob pro ni.
    Zobrazit řešení 
    // matematika.go:
func Nasob(a, b int) int { return a * b }

// matematika_test.go:
func TestNasob(t *testing.T) {
    if Nasob(3, 4) != 12 { t.Errorf("Nasob(3,4) != 12") }
    if Nasob(0, 5) != 0  { t.Errorf("Nasob(0,5) != 0") }
}
  2. Záměrně udělej chybu v Secti (return a - b místo a + b) a spusť testy — co uvidíš?
    Zobrazit řešení 
    Test TestSectiVice selže: "Secti(1, 2) = -1, chci 3"
a výpis bude červený FAIL.
  3. Spusť go test -cover ./... — co ti ukáže číslo coverage?
    Zobrazit řešení 
    Procento kódu pokrytého testy, např.
"coverage: 85.7% of statements"
