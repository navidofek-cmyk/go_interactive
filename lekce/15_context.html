⏱️ Lekce 15: Časový limit — context

⭐⭐⭐⭐⭐ Výzva! ⏱️ ~35 minut

🤔 Co je context?

context je jako časomíra ve hře nebo stopky ve škole. Řekneš: "Máš 2 sekundy na tah!" Pokud hráč nestihne, čas vyprší a hra pokračuje dál — bez čekání.

V Go umí context tři věci:

  • Timeout — "skonči do 2 sekund, jinak zruš"
  • Cancel — "zruš to hned teď!"
  • Hodnoty — předej informaci přes celý program (pokročilé)
⚽ context = píšťalka trenéra. Trenér řekne "Hraj!" (context) a po 90 minutách zapíská (cancel()). Všichni hráči (goroutiny) naslouchají ctx.Done() a ví, že mají zastavit.
Jak funguje context s timeoutem:
ctx, cancel := context.WithTimeout(...) ↓ goroutina 1 ←──── ctx.Done() ──── vypršel čas? → zastav! goroutina 2 ←──── ctx.Done() ──── vypršel čas? → zastav! ↓ cancel() ──────── okamžitě zruší všechny goroutiny
⏰ Sekce 1: WithTimeout — časový limit na stahování

💡 Jak to funguje?

Funkce stahniFoto používá select se dvěma kanály najednou. Buď doběhne práce (time.After) nebo vyprší čas (ctx.Done()) — co nastane dřív, to vyhraje. 

// Soubor: 15a_timeout.go
// Spusť: go run 15a_timeout.go

package main

import (
    "context"
    "fmt"
    "time"
)

// Simulujeme pomalou práci (třeba stahování ze sítě)
func stahniFoto(ctx context.Context, nazev string, delay time.Duration) error {
    fmt.Printf("📥 Stahuji: %s (bude trvat %v)\n", nazev, delay)

    select {
    case <-time.After(delay):    // práce dokončena
        fmt.Printf("✅ Staženo: %s\n", nazev)
        return nil
    case <-ctx.Done():           // čas vypršel!
        return fmt.Errorf("⏰ timeout: %s nestaženo (%v)", nazev, ctx.Err())
    }
}

func main() {
    fmt.Println("=== Stahujeme fotky (limit: 1 sekunda) ===\n")

    // Vytvoříme context s limitem 1 sekundy
    ctx, cancel := context.WithTimeout(context.Background(), 1*time.Second)
    defer cancel() // vždy uvolni zdroje!

    fotky := []struct {
        jmeno string
        cas   time.Duration
    }{
        {"kocka.jpg",      200 * time.Millisecond},
        {"pes.jpg",        500 * time.Millisecond},
        {"slon.jpg",       800 * time.Millisecond},
        {"dinosaurus.jpg", 1500 * time.Millisecond}, // příliš pomalé!
    }

    for _, f := range fotky {
        err := stahniFoto(ctx, f.jmeno, f.cas)
        if err != nil {
            fmt.Println("❌ Chyba:", err)
        }
    }

    fmt.Println("\nHotovo!")
}
Očekávaný výstup:
=== Stahujeme fotky (limit: 1 sekunda) === 📥 Stahuji: kocka.jpg (bude trvat 200ms) ✅ Staženo: kocka.jpg 📥 Stahuji: pes.jpg (bude trvat 500ms) ✅ Staženo: pes.jpg 📥 Stahuji: slon.jpg (bude trvat 800ms) ❌ Chyba: ⏰ timeout: slon.jpg nestaženo (context deadline exceeded) ❌ Chyba: ⏰ timeout: dinosaurus.jpg nestaženo (context deadline exceeded) Hotovo!

🧐 Proč slon.jpg selže?

Limit je 1 sekunda celkem. kocka.jpg (200 ms) a pes.jpg (500 ms) dohromady zaberou 700 ms — to stihnou. Potom zbývá jen 300 ms, ale slon.jpg potřebuje 800 ms — čas vyprší! A na dinosaurus.jpg nezbyde vůbec nic.

📯 Sekce 2: WithCancel — okamžité zrušení (fotbalový zápas)

💡 Rozdíl: Timeout vs Cancel

WithTimeout — automaticky se zruší po uplynutí doby.
WithCancel — zruší se až když ty sám zavoláš cancel(). To se hodí, když chceš zastavit goroutiny na základě nějaké události (třeba uživatel stiskne tlačítko "Stop"). 

// Soubor: 15b_cancel.go
// Spusť: go run 15b_cancel.go

package main

import (
    "context"
    "fmt"
    "time"
)

// Hráč hraje dokud ho trenér píšťalkou nezastaví
func hrac(ctx context.Context, jmeno string) {
    for {
        select {
        case <-ctx.Done():
            fmt.Printf("🛑 %s slyší píšťalku — zastavuje!\n", jmeno)
            return
        default:
            fmt.Printf("⚽ %s hraje...\n", jmeno)
            time.Sleep(400 * time.Millisecond)
        }
    }
}

func main() {
    fmt.Println("=== Fotbalový zápas ===\n")

    ctx, cancel := context.WithCancel(context.Background())

    // Spustíme hráče
    go hrac(ctx, "Tomáš")
    go hrac(ctx, "Marie")
    go hrac(ctx, "Karel")

    // Necháme hrát 1.2 sekundy
    time.Sleep(1200 * time.Millisecond)

    // Trenér píšťalkou zastaví všechny!
    fmt.Println("\n📯 Trenér píská — konec zápasu!")
    cancel()

    time.Sleep(200 * time.Millisecond) // počkáme na zastavení goroutin
    fmt.Println("\n🏆 Zápas skončil!")
}
Ukázkový výstup (pořadí hráčů se může lišit):
=== Fotbalový zápas === ⚽ Tomáš hraje... ⚽ Marie hraje... ⚽ Karel hraje... ⚽ Tomáš hraje... ⚽ Marie hraje... ⚽ Karel hraje... ⚽ Tomáš hraje... ... 📯 Trenér píská — konec zápasu! 🛑 Tomáš slyší píšťalku — zastavuje! 🛑 Marie slyší píšťalku — zastavuje! 🛑 Karel slyší píšťalku — zastavuje! 🏆 Zápas skončil!

📌 Důležité: vždy volej defer cancel()

Ať už použiješ WithTimeout nebo WithCancel, vždy ihned po vytvoření kontextu napiš defer cancel(). Tím uvolníš paměť a systémové zdroje, i když program skončí normálně. Je to dobrý zvyk!

  1. Změň timeout na 600ms — které fotky se stihnou stáhnout?
    Zobrazit řešení 
    ctx, cancel := context.WithTimeout(context.Background(), 600*time.Millisecond)
// Stihnou se jen kocka.jpg (200ms) a pes.jpg (500ms).
// Celkem 700ms > 600ms, takže pes.jpg ještě stihne (500ms < 600ms),
// ale slon.jpg (800ms) a dinosaurus.jpg (1500ms) nestihnou.
  2. Přidej 4. hráče "Eva" do fotbalového zápasu.
    Zobrazit řešení 
    // Přidej jeden řádek za go hrac(ctx, "Karel"):
go hrac(ctx, "Eva")
// Eva bude hrát a zastaví se spolu s ostatními po cancel().
  3. Změň čas zápasu ze 1.2s na 2s — kolik kol hráči odehrají?
    Zobrazit řešení 
    time.Sleep(2000 * time.Millisecond)
// Každý hráč hraje každých 400ms.
// Za 2 sekundy odehrají přibližně 5 kol (2000 / 400 = 5).
// Výpis bude delší — uvidíš víc řádků "hraje...".
