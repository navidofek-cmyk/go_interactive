🚒 Lekce 14: Chyby v týmu — errgroup

⭐⭐⭐⭐⭐ Výzva! ⏱️ ~35 minut

🤔 Co se naučíme?

Goroutina normálně nemůže "vrátit" chybu jako funkce — prostě skončí a nikdo se nic nedozví. Co když ale kuchař spálí dort? Jak to ostatní poznají?

Řešení: použijeme kanál pro chyby (error channel). Každý kuchař pošle výsledek (chybu nebo nil) do kanálu, a my je na konci sbíráme. Nebo použijeme errgroup z balíčku golang.org/x/sync — ale to je jen chytřejší wrapper okolo toho samého.

🍰 Představ si pekárnu: kuchaři vaří najednou. Jeden dort shoří. Kuchař zavolá "Spáleno!" do vysílačky (= error channel). Šéfkuchař uslyší a ví, kdo měl problém — zatímco ostatní kuchaři klidně dál pracují.
Jak funguje error channel:
Goroutina 1: vari() ──→ errCh ──→ chyba? nil Goroutina 2: vari() ──→ errCh ──→ chyba? "spáleno!" Goroutina 3: vari() ──→ errCh ──→ chyba? nil ↓ sbíráme chyby
📦 Sekce 1: Manuální error channel (pouze standardní knihovna)
// Soubor: 14a_error_channel.go
// Spusť: go run 14a_error_channel.go

package main

import (
    "fmt"
    "math/rand"
    "sync"
    "time"
)

// kuchař zkouší upéct dort — může se mu to pokazit
func upecDort(id int, nazev string, errCh chan<- error, wg *sync.WaitGroup) {
    defer wg.Done()

    fmt.Printf("👨‍🍳 Kuchař %d peče: %s\n", id, nazev)
    time.Sleep(300 * time.Millisecond)

    // 30% šance že dort shoří
    if rand.Float32() < 0.3 {
        chyba := fmt.Errorf("kuchař %d: %s shořel! 🔥", id, nazev)
        errCh <- chyba
        return
    }

    errCh <- nil // nil = vše ok
    fmt.Printf("✅ Kuchař %d dokončil: %s\n", id, nazev)
}

func main() {
    dorty := []string{
        "Čokoládový", "Jahodový", "Vanilkový",
        "Ovocný", "Karamelový",
    }

    errCh := make(chan error, len(dorty))
    var wg sync.WaitGroup

    fmt.Println("🏭 Pekárna spuštěna!\n")

    for i, dort := range dorty {
        wg.Add(1)
        go upecDort(i+1, dort, errCh, &wg)
    }

    wg.Wait()
    close(errCh)

    // Sbíráme výsledky
    fmt.Println("\n📋 Výsledky:")
    var chyby []error
    for err := range errCh {
        if err != nil {
            chyby = append(chyby, err)
        }
    }

    if len(chyby) == 0 {
        fmt.Println("🎉 Všechny dorty upečeny bez chyb!")
    } else {
        fmt.Printf("⚠️  %d dort(y) měly problémy:\n", len(chyby))
        for _, e := range chyby {
            fmt.Println(" ❌", e)
        }
    }
}
Ukázkový výstup (výsledek je náhodný — někdy 0 chyb, někdy více):

🚀 Sekce 2: errgroup — čistší způsob (vyžaduje modul)

💡 Co je errgroup?

errgroup je balíček z rodiny golang.org/x/sync. Dělá za tebe přesně to, co jsi napsal v sekci 1 — spravuje WaitGroup i chybový kanál najednou. Funkce eg.Go() spustí goroutinu a eg.Wait() počká na všechny a vrátí první chybu (nebo nil).

⚠️ Před spuštěním tohoto příkladu je potřeba inicializovat Go modul a stáhnout balíček:
go mod init pekarna
go get golang.org/x/sync/errgroup 
// Soubor: 14b_errgroup.go
// Nejprve spusť v terminálu:
//   go mod init pekarna
//   go get golang.org/x/sync/errgroup
// Pak: go run 14b_errgroup.go

package main

import (
    "fmt"
    "math/rand"
    "time"

    "golang.org/x/sync/errgroup"
)

func upecDortEG(id int, nazev string) error {
    fmt.Printf("👨‍🍳 Kuchař %d peče: %s\n", id, nazev)
    time.Sleep(300 * time.Millisecond)

    if rand.Float32() < 0.3 {
        return fmt.Errorf("kuchař %d: %s shořel! 🔥", id, nazev)
    }

    fmt.Printf("✅ Kuchař %d dokončil: %s\n", id, nazev)
    return nil
}

func main() {
    dorty := []string{"Čokoládový", "Jahodový", "Vanilkový", "Ovocný"}

    var eg errgroup.Group

    for i, dort := range dorty {
        i, dort := i, dort // zachytíme proměnné pro goroutinu
        eg.Go(func() error {
            return upecDortEG(i+1, dort)
        })
    }

    // Čekáme — vrátí první chybu nebo nil
    if err := eg.Wait(); err != nil {
        fmt.Println("\n❌ Nastala chyba:", err)
    } else {
        fmt.Println("\n🎉 Vše upečeno bez chyb!")
    }
}

🔍 Rozdíl mezi oběma přístupy

Error channel (sekce 1): sbírá všechny chyby — víš přesně, kdo selhal.
errgroup (sekce 2): vrátí jen první chybu a zbytek ignoruje — jednodušší kód, ale méně detailů. Pro většinu situací to stačí.

  2. Přidej výpis "🔄 Zkouším znovu!" když dort shoří, a zkus ho upéct podruhé (retry).
    Zobrazit řešení 
    func upecDort(id int, nazev string, errCh chan<- error, wg *sync.WaitGroup) {
    defer wg.Done()

    fmt.Printf("👨‍🍳 Kuchař %d peče: %s\n", id, nazev)
    time.Sleep(300 * time.Millisecond)

    if rand.Float32() < 0.3 {
        fmt.Printf("🔄 Zkouším znovu: %s\n", nazev)
        time.Sleep(300 * time.Millisecond)
        if rand.Float32() < 0.3 {
            errCh <- fmt.Errorf("kuchař %d: %s shořel podruhé! 🔥", id, nazev)
            return
        }
    }

    errCh <- nil
    fmt.Printf("✅ Kuchař %d dokončil: %s\n", id, nazev)
}
  3. Spočítej kolik kuchařů bylo úspěšných a kolik neúspěšných.

        chyby = append(chyby, err)
    } else {
        uspesnych++
    }
}
fmt.Printf("✅ Úspěšných: %d\n", uspesnych)
fmt.Printf("❌ Neúspěšných: %d\n", len(chyby))
